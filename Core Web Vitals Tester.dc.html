Likely cause
Synchronous stylesheet and script tags placed in the document <head>.
Estimated impact

Why it hurts
The oversized, legacy-format image dominates the LCP element and the total network payload.
Likely cause
Uncompressed JPEG served at 2× the display size with no responsive sizes.
Estimated impact
~0.9s faster LCP; roughly 1.1 MB saved.
Recommended fix
Convert to AVIF/WebP, generate a srcset, and lazy-load below-the-fold images.

Likely cause
Full library bundles imported instead of tree-shaken modules.
Estimated impact

Recommended fix

Why it hurts
Long tasks block user input and delay interactivity after the page appears.
Likely cause
Heavy JS execution and large style recalculations during hydration.
Estimated impact
~0.45s lower Total Blocking Time.
Recommended fix
Break up long tasks, memoize expensive work, and offload to web workers.

Why it hurts
Third-party code competes for the main thread and network bandwidth you don't control.
Likely cause
Analytics, chat widget, and A/B tooling all loaded eagerly at startup.
Estimated impact
~0.3s faster Total Blocking Time.
Recommended fix
Load third-parties on interaction, or behind a lightweight facade.
Affected resources
analytics 88 KB intercom 104 KB optimizely 48 KB
